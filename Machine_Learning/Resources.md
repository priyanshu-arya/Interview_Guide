# 📚 Machine Learning Recommended Resources

A hand-picked selection of top-tier learning resources for mastering Machine Learning, Deep Learning, MLOps, and LLMs for technical interviews.

---

## 📖 1. Official Documentation & Frameworks

| Resource | URL | Why It's Worth Using |
|:---|:---|:---|
| **PyTorch Documentation & Tutorials** | [pytorch.org/docs](https://pytorch.org/docs) | Essential for studying autograd mechanics, custom dataset implementations, distributed training (`torch.distributed`), and neural network modules. |
| **Hugging Face Transformers Docs** | [huggingface.co/docs](https://huggingface.co/docs) | The industry standard for pre-trained model loading, tokenization, PEFT/LoRA fine-tuning, and model evaluations. |
| **Scikit-Learn User Guide** | [scikit-learn.org](https://scikit-learn.org) | Unmatched clear explanations of classical ML algorithms, mathematical formulations, feature selection, and cross-validation pipelines. |
| **XGBoost & LightGBM Docs** | [xgboost.readthedocs.io](https://xgboost.readthedocs.io) | Detailed technical documentation on gradient boosting tree parameters, loss function optimization, GPU acceleration, and missing value handling. |

## 📘 2. Best Books

### 1. Classical Machine Learning & Intuition
- **"Hands-On Machine Learning with Scikit-Learn, Keras, and TensorFlow" (3rd Edition)** by *Aurélien Géron*
  - *Why Read*: The single best practical ML book. Excellent balance of code snippets, mathematical intuition, and end-to-end ML project workflows.

### 2. Deep Learning Theory
- **"Deep Learning"** by *Ian Goodfellow, Yoshua Bengio, and Aaron Courville*
  - *Why Read*: Known as the "Bible of Deep Learning". Essential for rigorous derivations of backpropagation, linear algebra, probability, regularizers, and generative modeling.

### 3. Machine Learning Systems & Design
- **"Designing Machine Learning Systems"** by *Chip Huyen*
  - *Why Read*: Indispensable for MLOps and System Design interviews. Covers data engineering, real-time feature stores, model serving latency, monitoring drift, and continuous deployment pipelines.

## 💻 4. Practice Websites & GitHub Repositories

- **LeetCode (Machine Learning Questions)**: Excellent for practicing NumPy matrix manipulation, custom metric code implementation, and basic classical ML algorithms.
- **Kaggle Discussions & Winning Kernels**: Study real-world feature engineering, ensembling strategies (stacking/blending), and hyperparameter tuning patterns from Grandmasters.
- **Papers With Code**: [paperswithcode.com](https://paperswithcode.com) — Track state-of-the-art (SOTA) benchmarks, datasets, and official open-source code implementations for modern ML papers.
- **Awesome-LLM-Interview**: GitHub repository collecting curated LLM engineering interview questions, RAG patterns, and fine-tuning guides.

## 🎓 5. Courses & Prep Platforms

1. **Stanford CS229: Machine Learning** (Andrew Ng) — Unmatched theoretical foundations in supervised/unsupervised learning, SVMs, and learning theory.
2. **Stanford CS224N: Natural Language Processing with Deep Learning** — Master word embeddings, RNNs, Self-Attention, and Transformer architectures.
3. **Fast.ai: Practical Deep Learning for Coders** (Jeremy Howard) — Top-down, code-first approach to deep learning, vision models, and fine-tuning.
