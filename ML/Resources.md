## 1. Official Documentation & Core Frameworks

- **[PyTorch Documentation](https://pytorch.org/docs/stable/index.html)**
  - *Why it's worth using*: Essential reference for deep learning primitives, autograd mechanics, distributed training (`torch.nn.parallel.DistributedDataParallel`), and tensor operations.
- **[Scikit-Learn User Guide](https://scikit-learn.org/stable/user_guide.html)**
  - *Why it's worth using*: The gold standard for understanding classical machine learning algorithms, preprocessing pipelines, model selection, and metrics.
- **[Hugging Face Transformers & PEFT Docs](https://huggingface.co/docs)**
  - *Why it's worth using*: Industry-standard documentation for fine-tuning LLMs, self-attention implementations, tokenizers, LoRA, and quantization.
- **[XGBoost Documentation](https://xgboost.readthedocs.io/)**
  - *Why it's worth using*: Deep dive into tree boosting algorithms, split finding mechanisms, regularized objective functions, and handling missing values.
- **[ONNX & ONNX Runtime](https://onnxruntime.ai/)**
  - *Why it's worth using*: Critical for understanding model serialization, cross-framework deployment, and low-latency inference optimization.

## 2. Seminal Books

- **[Pattern Recognition and Machine Learning (PRML) - Christopher Bishop](https://www.microsoft.com/en-us/research/people/cmbishop/prml-book/)**
  - *Why it's worth using*: Provides rigorous mathematical foundations for Bayesian methods, graphical models, maximum likelihood estimation, and linear models.
- **[Deep Learning - Ian Goodfellow, Yoshua Bengio, and Aaron Courville](https://www.deeplearningbook.org/)**
  - *Why it's worth using*: Known as the "Bible of Deep Learning". Covers optimization algorithms, convolutional networks, sequence models, autoencoders, and generative modeling.
- **[Hands-On Machine Learning with Scikit-Learn, Keras, and TensorFlow - Aurélien Géron](https://www.oreilly.com/library/view/hands-on-machine-learning/9781098125967/)**
  - *Why it's worth using*: Practical, project-driven guide explaining end-to-end ML workflows, data cleaning, feature engineering, and neural network training.
- **[Designing Machine Learning Systems - Chip Huyen](https://www.oreilly.com/library/view/designing-machine-learning/9781098107956/)**
  - *Why it's worth using*: Industry standard for ML System Design interviews. Focuses on data engineering, feature stores, model monitoring, continuous deployment, and data drift.

## 3. Top YouTube Channels & Video Courses

- **[Andrej Karpathy - Neural Networks: Zero to Hero](https://youtube.com/@AndrejKarpathy)**
  - *Why it's worth using*: Step-by-step implementation of Micrograd, Makemore, GPT-2, and tokenizers from pure Python/NumPy. Outstanding intuition builder for deep learning internals.
- **[3Blue1Brown - Neural Networks & Linear Algebra](https://www.youtube.com/@3blue1brown)**
  - *Why it's worth using*: Unrivaled visual animations explaining matrix multiplications, backpropagation, eigenvalues, and attention mechanisms.
- **[StatQuest with Josh Starmer](https://www.youtube.com/@statquest)**
  - *Why it's worth using*: Breaks down complex statistical ML algorithms (Random Forests, Gradient Boosting, SVMs, PCA, ROC curves) into simple step-by-step visual lessons.
- **[Stanford CS229: Machine Learning (Andrew Ng)](https://cs229.stanford.edu/)**
  - *Why it's worth using*: Benchmark academic course providing comprehensive mathematical derivations for supervised and unsupervised algorithms.
- **[Stanford CS224N: Natural Language Processing with Deep Learning](https://web.stanford.edu/class/cs224n/)**
  - *Why it's worth using*: Premier course for modern NLP, Word Embeddings, Transformers, Attention, and Large Language Models.

## 4. Practice Platforms & GitHub Repositories

- **[Kaggle Competitions & Datasets](https://www.kaggle.com/)**
  - *Why it's worth using*: Hands-on experience with feature engineering, ensembling, hyperparameter optimization, and real-world messy datasets.
- **[LeetCode Machine Learning / Data Science Questions](https://leetcode.com/)**
  - *Why it's worth using*: Excellent for practicing algorithm coding, matrix manipulations, and statistical coding under time constraints.
- **[Papers with Code](https://paperswithcode.com/)**
  - *Why it's worth using*: Links research papers directly to open-source implementations. Great for tracking state-of-the-art architectures and benchmarks.
- **[Applied-ML GitHub Repository (eugeneyan/applied-ml)](https://github.com/eugeneyan/applied-ml)**
  - *Why it's worth using*: Curated collection of production ML engineering papers and system architectures from tech giants (Netflix, Uber, DoorDash).

## 5. Blogs, Courses & Interview Prep Platforms

- **[Lil'Log (Lilian Weng)](https://lilianweng.github.io/)**

- **[Jay Alammar - The Illustrated Transformer / GPT-3](https://jalammar.github.io/)**
  - *Why it's worth using*: Highly visual explanations of Transformer components, Self-Attention, BERT, and GPT architectures.
- **[Chip Huyen's Blog](https://huyenchip.com/blog/)**
  - *Why it's worth using*: Insights into production ML, real-time machine learning, feature stores, and ML system design interview frameworks.
